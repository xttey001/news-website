## 四层分析体系

```
[原始搜索结果]
    ↓
┌─────────────────────────────┐
│  🐵 悟空 + 🐷 八戒（优质新闻）│  ← S级 + A级 + 抖音
│  输入：S/A级新闻            │
│  输出：市场情绪 + 贝叶斯决策  │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  🧅 沙僧（全部新闻）         │  ← 包括S/A/B级+噪音新闻
│  输入：7天全部新闻 × 时间衰减 │
│  输出：散户追高/恐慌概率      │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  🐴 白龙马（全部新闻+K线）   │
│  输入：新闻情绪 + K线 + 成交量│
│  输出：主力吸筹/出货/洗盘     │
└─────────────────────────────┘
    ↓
[news-data.js] → [GitHub Pages]
```

### 🐵 悟空 = 新闻分析师
- **输入**：S级 + A级 + 抖音优质新闻
- **职责**：不被表面新闻迷惑，挖掘信息背后的真实含义
- **风格**：找漏洞 / 辨机会 / 看后续 / 独立判断

### 🐷 八戒 = 概率分析师
- **输入**：悟空分析 + S/A级新闻
- **职责**：计算贝叶斯胜率、预期收益率、置信区间
- **输出**：最优ETF组合 + 止损线

### 🧅 沙僧 = 散户行为预测
- **输入**：**全部新闻**（含噪音、重复标题、B级新闻）
- **职责**：预测散户看到新闻后的行为
- **关键**：同类情绪叠加放大效应（7天累积 + 时间衰减）
- **输出**：追高买入概率 / 恐慌卖出概率 / TOP情绪触发源

### 🐴 白龙马 = 主力行为推断
- **输入**：全部新闻情绪 + 市场K线 + 成交量
- **职责**：结合价格走势推演主力意图
- **输出**：吸筹 / 拉升 / 出货 / 洗盘

## 核心原则

> **悟空/八戒处理优质新闻（决策辅助），沙僧/白龙马处理全部新闻（行为预测）。两者独立，互不干扰。**

- 沙僧看到的不是悟空筛选后的新闻，而是**原始的全部搜索结果**
- 多天累积 + 时间衰减 = 散户情绪的真实累积效应
- 重复标题 = 同类情绪强化信号

| 天数 | 权重 |
|------|------|
| 今天 | ×1.0 |
| 昨天 | ×0.7 |
| 前天 | ×0.5 |
| 3天前 | ×0.35 |
| 4天前 | ×0.25 |
| 5天前 | ×0.18 |
| 6天前 | ×0.13 |
| 长期逻辑（黄金/地缘） | ×1.0（无视衰减） |

## 白龙马主力状态判断

| 信号组合 | 主力判断 |
|---------|---------|
| 散户追高 + 价格涨 + 放量 | 主力可能在出货 |
| 散户恐慌 + 价格跌 | 主力可能在吸筹 |
| 缩量下跌 + 横盘 | 主力在洗盘 |
| 量价齐升 + 无FOMO | 主力拉升初期 |

## 三层交叉分析（核心设计）

> **白龙马分析悟空，八戒融合沙僧+白龙马。四层信号互相印证，矛盾时降低可信度。**

### 信号流向

```
悟空深度分析 ──────────────────────────┐
  (市场情绪)                           │
                                      ▼
白龙马 ──────────────────────→ 八戒最终结论
  (K线+成交量) ←───────────── ← (最优行动)
              (沙僧情绪)  ↑
                  (散户概率) ↑
              ←─────────────
沙僧
  (全量新闻)
```

### 第一层：白龙马融合悟空信号

白龙马在判断主力状态时，吸收悟空的情绪信号进行**矛盾检测**：

| 悟空情绪 | 白龙马判断 | 调整动作 | 可信度 |
|---------|-----------|---------|--------|
| 积极看多 | 主力出货 | 矛盾⚠️ | ×0.7 |
| 恐慌谨慎 | 主力出货 | 一致✅ | ×1.1 |
| 恐慌谨慎 | 主力吸筹 | 逆向机会📌 | ×1.2 |
| 积极乐观 | 主力吸筹 | 矛盾⚠️ | ×0.8 |

同时，白龙马也会吸收八戒的胜率信号：
- 八戒胜率>65% + 白龙马出货 → 双重矛盾，胜率降级
- 八戒胜率<50% + 白龙马吸筹 → 逆向布局机会

### 第二层：八戒融合沙僧+白龙马信号

**沙僧融合**：散户情绪影响胜率
- 散户狂热追高 + 八戒胜率>60% → **追高即被套**，胜率降至35%，止损线扩大至-12%
- 散户恐慌 → **逆向机会**，胜率+15%，止损缩小至-5%

**白龙马融合**：主力意图影响ETF推荐
- 主力出货 + 散户追高 → **坚决减仓回避**
- 主力吸筹 → **分批布局**（优先级最高）
- 信号矛盾 → **观望**

### 信号一致性评分

| 得分 | 含义 |
|------|------|
| 3分 | 四层信号共振：沙僧积极+白龙马拉升，或沙僧恐慌+白龙马吸筹 |
| 2分 | 信号正常：沙僧/白龙马无明显矛盾 |
| 1分 | 信号混乱：三方判断不一致，需谨慎 |

## 新闻分类标准

### S级新闻（🔴 主线级）
- **定义**：影响整个市场的重大新闻，影响1-4周
- **例子**：央行政策、地缘政治、美联储动向、重大行业政策
- **标记**：🔴 红色

### A级新闻（🟡 轮动级）
- **定义**：影响特定板块的重要新闻，影响1-3天
- **例子**：个股业绩、板块政策、突发事件
- **标记**：🟡 金色

### 抖音相关（📱 社交热点）
- **定义**：社交媒体热点、创业公司IPO、平台动态
- **标记**：📱 蓝色

### 操作类型说明
- **可做** → 用 ✅ 标记，机会明显
- **警惕** → 用 ⚠️ 标记，有风险需谨慎
- **回避** → 用 ❌ 标记，建议回避
- **关注** → 用 📌 标记，观察等待

### ETF sentiment 值
- `利好` → 红色背景
- `利空` → 绿色背景
- `中性` → 灰色

## 工作流程

### 四层处理流程

```
[1. 搜索新闻]
    ↓ 搜索10-20条全部新闻
[2. 保存 all_news]
    ↓ 全部保存到 all_news 字段
[3a. 筛选优质新闻]
    ↓ 从全部新闻中筛选S/A级
[3b. 悟空分析]
    ↓ 市场情绪 + 操作策略
[3c. 八戒贝叶斯分析]
    ↓ 每条S/A新闻的胜率 + 最终结论
[4. 运行 generate_full_newsdata.py]
    ↓ 沙僧模型处理 all_news → sangsha_module
    ↓ 白龙马模型处理 all_news + K线 → white_dragon
[5. 生成 news-data.js]
    ↓ 合并悟空/八戒 + 沙僧/白龙马
[6. git push + 飞书推送]
```

## 故障排除

### 网站没有更新
1. 确认 `news-data.js` 已更新
2. `git push origin main` 成功
3. 等待 1-3 分钟 GitHub Pages 构建
4. 清除浏览器缓存

### 网站显示空白/只有标题
**最可能原因**：`index.html` 的 JavaScript 代码有语法错误

**排查步骤**：
1. 用浏览器开发者工具（F12）→ Console 查看是否有红色报错
2. 常见错误：
   - `ReferenceError: xxx is not defined` → 变量作用域问题
   - `Unexpected token` → JSON/JS 语法错误
   - `Uncaught SyntaxError` → 括号/花括号不匹配

**预防措施（必须遵守）**：
1. **编辑 index.html 后立即检查语法**
2. **git diff 审查**：`git diff --cached index.html` 确保只有预期改动
3. **禁止复制粘贴整个代码块**：只编辑需要的部分，不要大段复制
4. **const 变量作用域检查**：确保 `const` 声明在正确的作用域内

### 历史问题记录

**2026-04-03 空白页面 Bug**：
- 原因：编辑时误将「悟空判断」渲染代码复制了两次，第二次放在了「八戒贝叶斯」之后，导致变量在作用域外被引用
- 表现：页面只显示标题和 footer，所有新闻内容消失
- 修复：删除重复的代码块
- 教训：大段代码编辑必须用 `git diff` 审查，推送前验证

**2026-04-04 CSS 类名不匹配 Bug**：
- 原因：CSS 中定义的类名（`.wukong-item`）与 HTML 中使用的类名（`.judgment-item`）不匹配
- 表现：悟空判断和决策矩阵样式错乱
- 修复：统一 CSS 类名与 HTML 结构
- 教训：添加新功能时必须确保 CSS 和 HTML 类名一致

**2026-04-05 中文引号导致 JSON 解析失败**：
- 原因：`news-data.js` 中的 summary 字段包含中文引号 `"` `"`，这些字符在 JavaScript/JSON 字符串中被识别为字符串终止符，导致 `SyntaxError: Invalid or unexpected token`
- 表现：网站加载后只显示标题和日期选择器，所有新闻内容消失；浏览器控制台报错 `Unexpected token`
- 修复：将中文引号 `"` `"` 替换为中文书名号 `「` `」` 或英文引号 `\"`
- 教训：**写入 news-data.js 前必须检查中文引号**

### ⚠️ news-data.js 写入前检查清单（强制）

**每次写入 `news-data.js` 之前，必须执行以下检查：**

```bash
# 方法1: 使用自动检查脚本（推荐）
node C:\Users\asus\.qclaw\workspace\skills\finance-news-dashboard\scripts\check-json.js "C:\Users\asus\.qclaw\workspace\news-server\github-pages-deploy\news-data.js"

# 方法2: 手动检查
# 2.1 语法检查
node --check news-data.js

# 2.2 中文引号检查（如果有输出则有问题）
Select-String -Path news-data.js -Pattern '[""'']' | Select-Object -First 5
```

**禁止使用的字符（在 JSON 字符串值中）**：
| 字符 | 名称 | 替代方案 |
|------|------|---------|
| `"` | 中文左双引号 | 使用 `「` 或 `\"` |
| `"` | 中文右双引号 | 使用 `」` 或 `\"` |
| `'` | 中文左单引号 | 使用 `『` 或 `\'` |
| `'` | 中文右单引号 | 使用 `』` 或 `\'` |

**正确做法**：
- ✅ `"summary": "伊朗军方强硬回应特朗普「48小时」通牒"`
- ✅ `"summary": "伊朗军方强硬回应特朗普\"48小时\"通牒"`
- ❌ `"summary": "伊朗军方强硬回应特朗普"48小时"通牒"` ← 会导致语法错误
