## 📅 每日更新（3 种方式）

### 方式 1：手动更新（最简单）

```powershell

```

**耗时**：~30 秒
**适用**：偶尔更新

---

### 方式 2：手动步骤（最灵活）

```powershell
cd C:\Users\asus\.qclaw\workspace\news-server

python generate_static_with_history.py

copy index.html github-pages-deploy\

# 推送到 GitHub

git add .

git push
```

**耗时**：~1 分钟
**适用**：需要自定义的情况

---

### 方式 3：自动定时更新（最省心）

```powershell
cd C:\Users\asus\.qclaw\workspace\news-server
python setup_daily_cron.py
openclaw gateway restart
```

**耗时**：设置一次，之后自动
**适用**：每天固定时间更新
**时间**：每天 23:00 自动更新

## ⚠️ 注意事项

1. **GitHub 推送需要认证**
   - 第一次推送需要输入 GitHub 用户名和密码
   - 或使用 Personal Access Token

2. **网站更新延迟**
   - GitHub Pages 通常 1-3 分钟更新
   - 可能需要刷新浏览器缓存（Ctrl+Shift+R）

3. **历史数据保留**
   - 默认保留最近 30 天
   - 可在 `daily_update.py` 中修改

4. **新闻数据格式**
   - 必须是 `news_YYYY-MM-DD.json` 格式
   - 必须包含 `date`, `market_tone`, `s_level`, `a_level`, `douyin` 字段

## 🆘 故障排查

### 问题：推送失败

**解决**：
```powershell
cd github-pages-deploy
git status  # 检查状态
git log     # 查看历史
```

### 问题：网站不更新

**解决**：
1. 检查 GitHub 仓库是否收到推送
2. 检查 GitHub Pages 设置
3. 清除浏览器缓存（Ctrl+Shift+R）

### 问题：日期显示错误

**解决**：
- 检查 `news_YYYY-MM-DD.json` 文件名格式
- 检查 JSON 文件是否有效
