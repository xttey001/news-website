## 关键教训

### 1. availableDates 必须同步更新
每次添加新日期数据时，必须同时更新 `availableDates` 数组：
```javascript
// 错误做法：只更新 newsData
newsData["2026-04-21"] = newData;

// 正确做法：同时更新 availableDates
newsData["2026-04-21"] = newData;
if (!availableDates.includes("2026-04-21")) {
  availableDates.unshift("2026-04-21");
}
```

### 2. timeout 要留余量
- 公式: `timeout = 预估耗时 × 1.5`
- 五层分析任务建议: 900秒（15分钟）

### 3. 双重验证
- Git push 成功 ≠ 网站显示正确
- 需要验证: `newsData` + `availableDates` + 网站实际显示

### 4. 缓存延迟
- GitHub raw 文件缓存: 1-2分钟
- GitHub Pages 重建: 1-3分钟
- 用户端需要 Ctrl+Shift+R 强制刷新
